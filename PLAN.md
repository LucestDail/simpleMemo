# SimpleMemo — 상세 구현 계획

## 1. 프로젝트 비전

저스펙 장비를 활용한 네트워크 기반 메모 작성 및 기록 관리 프로그램. 내부망에서 간편하게 사용할 수 있는 디지털 블랙보드.

## 2. 현재 상태

- Express + Socket.io 백엔드 + Vue 3 + Vite 프론트엔드
- 실시간 동기화 (Socket.io)
- 포스트잇 (드래그, 색상, 내용)
- JSON 파일 기반 저장 (`data/memo.json`)
- Docker 지원
- 디바운스 불일치 (README 500ms vs 코드 100ms)
- 멀티 탭 데이터 모델은 있으나 UI 미구현
- 레거시 `public/` 폴백 코드 잔존

## 3. 디자인 시스템

**Google Material Design 3 (M3) 준용**

### M3 적용 방침
Vue 3 프로젝트이므로 Vuetify 3 경량 도입 또는 Tailwind + M3 원칙 조합:

### 컬러 시스템
| 토큰 | Light | Dark | 용도 |
|------|-------|------|------|
| Primary | `#6750A4` | `#D0BCFF` | 주요 액션, 강조 |
| On Primary | `#FFFFFF` | `#381E72` | Primary 위 텍스트 |
| Secondary | `#625B71` | `#CCC2DC` | 보조 요소 |
| Tertiary | `#7D5260` | `#EFB8C8` | 포스트잇 강조 |
| Surface | `#FEF7FF` | `#1D1B20` | 보드 배경 |
| Error | `#B3261E` | `#F2B8B5` | 삭제/경고 |

> M3 기본 팔레트 (Baseline) 활용 — 범용적이고 친근한 퍼플 계열

### 포스트잇 컬러 팔레트
| 이름 | 값 | 용도 |
|------|-----|------|
| Yellow | `#FFF9C4` | 일반 메모 |
| Pink | `#F8BBD0` | 중요 |
| Blue | `#BBDEFB` | 정보 |
| Green | `#C8E6C9` | 완료/확인 |
| Orange | `#FFE0B2` | 주의 |
| Purple | `#E1BEE7` | 아이디어 |

### 타이포그래피
- 본문: Pretendard / Noto Sans KR
- M3 Type Scale 적용
- 보드 뷰에서는 가독성 우선 (큰 폰트)

### 핵심 컴포넌트
- 포스트잇: M3 Elevated Card + 그림자 + 컬러
- 보드: M3 Surface 배경
- 입력: M3 Text Field (Filled variant)
- 메뉴: M3 Bottom App Bar (모바일)
- `border-radius: 16px` (포스트잇 큰 라운드)

## 4. 단계별 구현 계획

### Phase 1 — 기존 이슈 수정 + 정리 (2주)

**1.1 코드 정합성**
- [ ] 디바운스 통일 — 코드(100ms)와 README(500ms) → 300ms로 통일 (사용자 설정 가능)
- [ ] 레거시 `public/` 폴백 코드 제거 — `dist/` 빌드를 필수로 변경
- [ ] `server.js`의 기본 메모 데이터 구조 정리

**1.2 멀티 탭 UI 구현**
- [ ] 데이터 모델에 이미 정의된 탭(`general`, `todo`, `links`, `ideas`) 활성화
- [ ] 탭 전환 UI (M3 Tab Bar)
- [ ] 탭별 독립 포스트잇 관리
- [ ] 탭 추가/삭제/이름변경 (선택적)

**1.3 Socket.io 안정성**
- [ ] 연결 끊김 시 자동 재연결 + 큐잉
- [ ] 오프라인 변경 → 재연결 시 동기화
- [ ] 연결 상태 표시 (상단 배너)

### Phase 2 — 포스트잇 기능 확장 (3주)

**2.1 포스트잇 UI 개선**
- [ ] M3 Elevated Card 스타일 적용
- [ ] 색상 팔레트 확장 (6가지 프리셋)
- [ ] 크기 조절 (드래그로 리사이즈)
- [ ] 핀 고정 (이동 잠금)
- [ ] 레이어 순서 (앞으로/뒤로)
- [ ] 최소화 (제목만 표시)

**2.2 콘텐츠 확장**
- [ ] 체크리스트 모드 — 포스트잇 내 체크박스 목록
- [ ] 이미지 첨부 — 드래그 앤 드롭 / 클립보드 붙여넣기
- [ ] 타임스탬프 — 생성/수정 시간 자동 기록 + 표시
- [ ] 우선순위 표시 — 아이콘 또는 테두리 색상

**2.3 멀티 보드**
- [ ] 여러 보드 생성 (회의실, 부엌, 거실, 개인 등)
- [ ] 보드 전환 UI
- [ ] 보드별 독립 `memo.json` (또는 통합 JSON 내 보드 키)
- [ ] QR코드 접속 — 보드별 QR 생성, 스마트폰으로 스캔하여 접속

### Phase 3 — 검색 + 아카이브 + 백업 (2주)

**3.1 메모 검색**
- [ ] 전체 보드/포스트잇에서 키워드 검색
- [ ] 검색 결과 하이라이트
- [ ] 태그 기반 필터

**3.2 아카이브**
- [ ] 오래된 메모를 삭제하지 않고 아카이브로 이동
- [ ] 아카이브 목록 조회
- [ ] 복원 기능

**3.3 데이터 백업**
- [ ] 자동 백업 — 주기적으로 `data/memo.json` 스냅샷 저장 (최근 10개)
- [ ] 수동 export/import (JSON 파일 다운로드/업로드)
- [ ] 백업 복원 UI

### Phase 4 — M3 UI + 안정화 (2주)

**4.1 전체 UI M3 전환**
- [ ] Controller.vue: M3 입력 필드, 버튼, 탭
- [ ] Board.vue: M3 Surface 배경, 포스트잇 카드
- [ ] 다크/라이트 모드 토글
- [ ] 반응형: 모바일(컨트롤러), 데스크톱(보드 전체화면)

**4.2 접근성**
- [ ] 보드 URL을 QR코드로 표시하는 오버레이 버튼
- [ ] 글꼴 크기 조절 (보드 표시용)
- [ ] 전체화면 모드 (F11 또는 버튼)

**4.3 테스트**
- [ ] Socket.io 동시 접속 테스트 (10+ 기기)
- [ ] 데이터 충돌 해결 테스트
- [ ] 저스펙 기기 성능 테스트 (Raspberry Pi 등)
