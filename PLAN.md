# SimpleMemo — 상세 구현 계획

## 1. 프로젝트 비전

저스펙 장비를 활용한 네트워크 기반 메모 작성 및 기록 관리 프로그램. 내부망에서 간편하게 사용할 수 있는 디지털 블랙보드.

## 2. 현재 상태

- Express + Socket.io 백엔드 + Vue 3 + Vite 프론트엔드
- 실시간 동기화 (Socket.io)
- 포스트잇 (드래그, 색상, 내용)
- JSON 파일 기반 저장 (`data/memo.json`)
- Docker 지원
- 디바운스 불일치 (README 500ms vs 코드 100ms)
- 멀티 탭 데이터 모델은 있으나 UI 미구현
- 레거시 `public/` 폴백 코드 잔존

## 3. 디자인 시스템

## 4. 단계별 구현 계획

### Phase 1 — 기존 이슈 수정 + 정리 (2주)

**1.1 코드 정합성**
- [ ] 디바운스 통일 — 코드(100ms)와 README(500ms) → 300ms로 통일 (사용자 설정 가능)
- [ ] 레거시 `public/` 폴백 코드 제거 — `dist/` 빌드를 필수로 변경
- [ ] `server.js`의 기본 메모 데이터 구조 정리

**1.2 멀티 탭 UI 구현**
- [ ] 데이터 모델에 이미 정의된 탭(`general`, `todo`, `links`, `ideas`) 활성화
- [ ] 탭 전환 UI (M3 Tab Bar)
- [ ] 탭별 독립 포스트잇 관리
- [ ] 탭 추가/삭제/이름변경 (선택적)

**1.3 Socket.io 안정성**
- [ ] 연결 끊김 시 자동 재연결 + 큐잉
- [ ] 오프라인 변경 → 재연결 시 동기화
- [ ] 연결 상태 표시 (상단 배너)

### Phase 2 — 포스트잇 기능 확장 (3주)

**2.1 포스트잇 UI 개선**
- [ ] M3 Elevated Card 스타일 적용
- [ ] 색상 팔레트 확장 (6가지 프리셋)
- [ ] 크기 조절 (드래그로 리사이즈)
- [ ] 핀 고정 (이동 잠금)
- [ ] 레이어 순서 (앞으로/뒤로)
- [ ] 최소화 (제목만 표시)

**2.2 콘텐츠 확장**
- [ ] 체크리스트 모드 — 포스트잇 내 체크박스 목록
- [ ] 이미지 첨부 — 드래그 앤 드롭 / 클립보드 붙여넣기
- [ ] 타임스탬프 — 생성/수정 시간 자동 기록 + 표시
- [ ] 우선순위 표시 — 아이콘 또는 테두리 색상

**2.3 멀티 보드**
- [ ] 여러 보드 생성 (회의실, 부엌, 거실, 개인 등)
- [ ] 보드 전환 UI
- [ ] 보드별 독립 `memo.json` (또는 통합 JSON 내 보드 키)
- [ ] QR코드 접속 — 보드별 QR 생성, 스마트폰으로 스캔하여 접속

### Phase 3 — 검색 + 아카이브 + 백업 (2주)

**3.1 메모 검색**
- [ ] 전체 보드/포스트잇에서 키워드 검색

- [ ] 태그 기반 필터

### Phase 5 — AI 기능 도입 (예정)

**원칙**
- AI는 **제안만**, 사용자가 수락해야 반영됨 (자동 덮어쓰기 없음)
- 키는 서버 `.env`에만 존재, 프런트는 프록시만 호출
- 실패 시 조용히 비활성, 본 앱 흐름 방해 금지
- 작은 `✨` 버튼 1개 수준으로만 UI 침투

**5.0 기반 (Phase 0)**
- [ ] `.env`: `AI_ENABLED`, `AI_PROVIDER=openai`, `AI_API_KEY`, `AI_MODEL=gpt-4o-mini`, `AI_BASE_URL`, `AI_MONTHLY_TOKEN_LIMIT`
- [ ] `server/ai/` 모듈: `provider.js`(인터페이스) / `openai.js`(어댑터) / `cache.js`(LRU) / `guard.js`(IP rate limit + 월 토큰 카운터)
- [ ] 단일 라우트 `POST /api/ai/suggest` (task 분기: `meta | summary | todos | qa`)
- [ ] JSON 모드 강제(`response_format: { type: "json_object" }`), 5s 타임아웃
- [ ] 프런트 `composables/useAi.js` — 서버만 바라보는 얇은 래퍼
- [ ] Settings에 "AI 사용" 토글 + 연결 상태 뱃지 + 개인정보 전송 고지 1줄

**5.1 체감 효용 큰 기능부터 (Phase 1)**
- [ ] **제목·탭·색상 자동 제안** — 본문 ≥ 2줄 + `✨` 칩 클릭 → `{title, tab, color}` 수락 UI
- [ ] **요약 / 재작성** — 포스트잇 편집 모드 `✨ 요약` → 미리보기 + `[적용]`

**5.2 쓰기 흐름 강화 (Phase 2)**
- [ ] **체크리스트 추출** — 자유 메모 → `{text, done}[]`, `checklist` 필드에 세팅
- [ ] **유사 메모 경고** — 새 메모 추가 시 최근 항목과 유사도 검사 (키워드 → Phase 3에서 임베딩으로 교체)

**5.3 검색·회고 (Phase 3)**
- [ ] **자연어 검색 / Q&A** — 검색창 프리픽스 `?`
- [ ] **임베딩 인덱스** — `text-embedding-3-small`, `data/embeddings.json` 변경분만 증분 계산
- [ ] **주간·일간 리뷰 카드** — 기간 선택 요약 위젯

**5.4 멀티모달 (Phase 4, 선택)**
- [ ] **음성 → 텍스트** (Whisper)
- [ ] **이미지 OCR** (포스트잇 사진 → 텍스트)

**보안·비용·운영 체크리스트**
- [ ] 키는 서버 `.env`만, `localStorage`·번들 금지
- [ ] `POST /api/ai/*` per-IP rate limit (예: 10 req/min)
- [ ] 요청 본문 최대 4KB / 500토큰 입력 상한
- [ ] 월 토큰 카운터 초과 시 자동 OFF + Settings 경고
- [ ] 응답 지연 5s 초과 시 조용히 fallback
- [ ] Settings에 "AI 사용 시 OpenAI로 메모 내용이 전송됩니다" 고지
- [ ] LAN 운영 환경: 아웃바운드 443 허용 여부 확인

**구현 착수 전 확정할 항목**
1. 1차 기능 범위: 5.1 둘 다? 또는 제목 제안만?
2. UI 노출: `✨` 버튼만 vs 타이핑 멈춤 시 자동 제안
3. 기본 모델: `gpt-4o-mini` 확정 여부
4. 월 토큰 상한(또는 호출 상한) 숫자
